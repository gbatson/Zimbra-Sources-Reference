Selecting Users to Migrate
1.
In the Users dialog, select the users to migrate using the Population Tools.
•
Load CSV: Click Load CSV to use a CSV file, such as an Excel spreadsheet, that lists the provisioned user accounts and their passwords to use for migration.This allows for mapping email addresses between the Exchange server and the Zimbra server. If you use mapping, the source name is mapped to the destination name.
•
Object Picker: Click Object Picker utilized the Exchange server to find users. Select users, computers, and groups, and the location to search. You can also enter the object name to select.
•
Select this object type: Click Object Types to select the types of object you want to find, such as computers, groups, or users.
•
From this location: Click Locations to select the domain from which to select the accounts.
•
Enter the object names to select: In the Enter object names text box, type the user names or partial user names and click Check Names. You can search for multiple objects by using object names and separating each name with a semicolon. For example, if you type Jo, all names that begin with Jo are displayed, i.e., Joe Smith, Joseph Flanders, Joanna Writh. Use one of the following syntax examples:
•
DisplayName (example: FirstName LastName)
•
ObjectName (example: Computer1)
•
UserName (example: User1)
•
ObjectName@DomainName (example: User1@Domain1)
•
DomainName\ObjectName (example: Domain1\User1)
Click Advanced to specify a more detailed search.
When the object name has been typed, click OK. The source and destination mailboxes dialog displays with your selections listed.
•
LDAP Browser: Click LDAP Browser to select groups of accounts to migrate. In the LDAP Browser dialog, click Browse to select the base distinguished name (DN) of the container to be used as the root for the search. If you want all folders under the selected DN base to be included in the migration, check Search entire subtree. Create a filter to identify which mailboxes to import. Click OK. The Source Name and Destination Name mailboxes dialog refreshes with your selections listed in the box.
•
Add: Click Add to manually add a user on the Exchange server to be migrated to the Zimbra server.
•
Remove: Click Remove to manually remove a user on the Exchange server to be migrated to the Zimbra server.
•
Import users into domain. Select the destination domain to migrate users.
•
Save CSV. When the user selection is complete, the wizard generates a .csv file with a list of new accounts. You should download this file for future reference. Choose a secure location to store the file as it may contain password information for the user accounts you provision.
